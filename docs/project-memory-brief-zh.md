# Starry No Worry 简版记忆

1. 项目 `starry_no_worry` 的长期目标是全功能占星排盘平台，当前按阶段递增开发。
2. 第一阶段是可上线的占星工具型 MVP，第二阶段重点是专业计算能力、更多盘型、后续 AI 解读。
3. 前端是 React/Vite，后端优先，使用 Python/FastAPI，数据保存目前保持轻量。
4. 视觉主题是春天配色，但要兼顾专业占星识别度与普通用户可读性。
5. 前端表格负责精确阅读，星盘可视化负责空间感知。
6. 已打通本命盘、比较盘、流年盘的基础前后端能力。
7. 后端已支持宫位、ASC、MC、经纬度、时区细化，并已切换到 `pyswisseph`。
8. 已支持南北交、凯龙、莉莉丝、福点、宿命点，以及多宫制、相位集合、容许度配置。
9. 前端已有较成熟的星盘可视化：单层/双层轮盘、中文化、tooltip、表格联动思路。
10. 第二阶段新增重点主线是扩展更多星盘类型：组合盘、Davison、马盘、日返、月返、三限相关变体。
11. 第二阶段推荐顺序：公共计算能力 → 共用盘型生成框架 → 新盘型落地 → 分析交互 → AI 解读增强。
12. 后端已开始统一生成框架：单体衍生、双体比较、双体融合三类生成器方向。
13. `transit` 已迁移到单体衍生框架，`synastry` 已迁移到双体比较框架。
14. 中文术语必须严格统一：人名用用户输入，行星名、星座名、相位/宫位/overlay 表述都要专业且精确。
15. 用户非常在意前端细节，尤其是 tooltip、相位描述、轮盘标注和表格可读性。
16. backend 测试环境以 `.venv312` 为准；当前 backend 全量测试为 `54 passed`。
17. 子代理命名必须带功能目标，方便人工审阅，例如：`impl_transit_generator`、`review_synastry_adapter`、`verify_backend_chart_suite`。
