# Task Plan Template

Create one execution-ready plan per file under `work/plans/`.

## Header

- task name
- date
- status
- phase

## Sections

1. task brief
2. product intent
3. scope
4. acceptance criteria
5. constraints
6. execution graph
7. verification plan
8. risks
9. recovery rules
10. decision capture

For long rollout plans, the execution graph should contain ordered steps or slides, and only one should be current at a time.
