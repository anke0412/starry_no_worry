# Task Plan Template

Create one execution-ready task per file under `work/plans/`.

## Header

- Task name
- Date
- Status
- Phase

## 1. Task Brief

- goal
- why now

## 2. Product Intent

- user or product value

## 3. Scope

- in scope
- out of scope
- protected files or areas

## 4. Acceptance Criteria

- observable outcomes
- contract or UX expectations

## 5. Constraints

- roadmap constraints
- architecture constraints
- terminology or polish constraints

## 6. Execution Graph

- main-agent work
- `impl` subagent work
- `review` subagent work
- `verify` subagent work

## 7. Verification Plan

- backend tests
- frontend tests
- build
- targeted checks

## 8. Risks

- product risks
- technical risks
- coordination risks

## 9. Recovery Rules

- when to replan
- when to redispatch
- when to escalate

## 10. Decision Capture

- what should land in `work/decisions/`
- what should be promoted back into stable docs
