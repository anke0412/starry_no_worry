# Harness Reuse Guide

This repo is already an adopted project rather than a blank starter. Reuse within this repo means:

- stable truths go under layered `docs/`
- active execution goes under `work/`
- the main roadmap remains `plan.md`

If future repos copy this structure, they should adapt `project-bootstrap.md`, the layered docs, and the runtime `work/` conventions to their own roadmap.
