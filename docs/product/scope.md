# Product Scope

## Current In-Scope Direction

- extend the astrology chart system through staged delivery
- support single-person, two-person, and timing-related chart workflows
- keep interpretation deterministic or structured unless a later phase explicitly upgrades it
- improve analysis and interaction only after chart generation outputs are stable

## Current Phase 2 Focus

- expand advanced chart families
- reuse the shared chart generation framework
- preserve frontend readability and terminology quality as new chart types arrive

## Explicit Non-Goals Right Now

- broad account platform work
- payment or community features
- replacing chart math with LLM behavior
- speculative feature expansion that bypasses the Phase 2 order

## Freeze Rules

- do not casually rewrite the product roadmap in `plan.md`
- do not skip shared-generation reuse just to ship a new chart type faster
